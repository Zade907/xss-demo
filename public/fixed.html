Why Input Validation Is the
First Line of Defence

By Priya Sharma  ·  January 15, 2025

Every web application trusts its users to some degree — but how much trust is too much? The history of web security is littered with examples of developers who assumed their users would behave, only to discover that a single malformed input could bring down an entire system.

Cross-Site Scripting, or XSS, remains one of the most prevalent vulnerabilities on the web. The fix is simple: never render user-controlled content as HTML. Use textContent, not innerHTML. Encode output at the point of rendering.

⚠ Vulnerable (index.html)
✓ Fixed (this page)
div.innerHTML =
comment.body;
// executes scripts
div.textContent =
comment.body;
// displays as text

Submit the same XSS payload on this page — you'll see it appear in the comment section as literal text rather than executing. The injected <script> tag is treated as a string, not as markup.
